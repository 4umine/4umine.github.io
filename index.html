vmware 安装 centos7

发表于 2016-11-05   |   分类于 linux   |  

最近在看《大型网站技术架构》这本书，有一种相见恨晚的感觉。虽然现在的自己对网站架构概念还比较模糊，但是平常逛大牛博客对一些名词也有所耳闻，例如分布式系统、负载均衡、反向代理和消息队列等等。
所以脑子中总会冒一些问题，到底啥事分布式，服务器为啥要集群，网站大了以后要怎么扩展等系列问题。很幸运的是，这本书开头就讲了大型网站架构演化的过程，有种茅塞顿开之感。既然有心向之，那就do it!
第一步安装linux（卧槽，才入正题，废话真多~），笔者选用的centos7，网上安装教程一大堆，但还是结合自己的总结下。

下载 vmware 和 centos7

vmware pro 下载

centos7 下载

安装centos（一）

在安装完vmware以后，新建虚拟机，默认典型，下一步

选择“稍后安装操作系统”

客户机操作系统选择 linux，版本选择CentOS 64位

选择安装位置，例如我的vm_01, vm_02, vm_03…

磁盘容量默认20G，选择“将磁盘存储为单个文件”

自定义硬件用默认的就行，点击“完成”

安装centos（二）

右键刚创建好的虚拟机，点“设置”，单击“CD/DVD (IDE)”，选择“使用 ISO镜像文件”，浏览下好的centos操作系统的iso文件，点“确定”

点击“开启此虚拟机”，在出现的界面中选择 “Install CentOS 7”（注：将输入定位到虚拟机，按Ctrl + G；回到原操作系统，按Ctrl + Alt）

如果你一切还顺利的话，就到了选择语言的界面

点击“日期和时间”，自己设置下就行

点击“安装源”，默认选中“自动检测到的安装介质”，不用改，点“完成”

点击“软件选择”，我选的是“虚拟化主机”，因为我只想用命令行，所以不用客户端，“最小安装”好多功能要自己另外配，麻烦

点击“安装位置”，点击下面的“我要配置分区”，点“完成”，出现的界面中选择“标准分区”

点击左下角“+”，添加/boot，大小为200M

并设置”文件系统“为 ext4

继续添加挂载点swap，容量为2G，不用修改文件系统类型

添加/home，大小2G，文件系统为ext4

添加/，容量不填，即剩余容量都分配给它，文件系统为ext4

点击”完成“

点击”接受更改“

点击”网络和主机名“，开启，点击”完成“

点击”开始安装“

安装要个几分钟，这时候你可以设置root密码

设置root账户密码

安装完成后”重启“

如果没问题系统会提示登陆，输入root账户名和密码

登陆成功

到这vmware中安装centos就搞定了！

eclipse项目迁移到Intellij idea踩到编译器的坑

发表于 2016-03-18   |   分类于 code   |  

近期尝试使用idea，把公司项目从github上检出到idea中，设置一番，启动tomcat。一切看起来貌似很顺利，但是事实总比你想象的要糟糕，启动报错了。推断的类型不符合声明的范围（前提项目本身没问题，公司一直用这么一套框架来着）。在设置各种jdk版本后，依然不好使。最后想到可能是编译环境的问题，找到compile

此处显示use compiler，由于肚子里还剩几个单词，当然知道这里是选择编译器。但为什么要选择编译器，默认是javac，就是当初学java的时候用window命令行javac xx编译java文件的那个编译器。说实话，到现在我也就认识这个-_-。展开选择，发现有Eclipse, 至于下面两个是啥就真不知道，管它呢。选择eclipse试试。果然，tomcat成功启动，chrome弹出项目首页。此时，心里这个爽啊！！！

但是为啥javac和eclipse这两个编译器有区别呢？为啥还有这么多的java编译器呢？这个编译器是只对javac包了层壳还是真的就是两种不同的compiler？于是Google了下，在爆栈上大致了解了下what is the difference between javac and the eclipse compiler
主要两个意思：

  1. eclipse实现了自己的编译器，叫Eclipse Compiler for Java（ECJ）
  2. ECJ优于javac，其中有一个就是我碰到的，如果你的代码块有error（但确实没错），javac never run，但是ECJ run fine.
