Cookie 那点事

Mar 22, 2016 | code | Hits

今天在给以前的项目做迭代发现代码中的小问题，先看代码

代码的意思就是先存cookie，再取cookie。那么问题来了，从request中取cookie能取到值么？在上面代码样例中肯定是不能的，通过response设置cookie到客户端，前提是response要提交，也就是浏览器要成功收到响应。在这个代码中，设置cookie的同时又去获取cookie值，city永远是null（除非第二次访问这个方法）。

为什么我对这段代码这么敏感，因为之前自己也犯这样的错，而且这个错还特别不好调试，因为它昙花一现。比如写完代码开始打开浏览器访问，虽然第一次city是为null。但一般这时候你不会打断点去调试，一心想着页面能不能正常显示（人的想象总是美好的）。接着你就要调试调试，再次访问发现一切又正常了，city也有值了。于是你就说eclipse抽风了，不断调试，发现没啥问题，可能人品不好吧，你只能这么说。某一天，cookie被清，问题重现了，准备调试了，又tm正常了！然后，你疯了。

吃一堑，长一智。
